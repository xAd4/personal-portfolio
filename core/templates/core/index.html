1. ¿Qué es una API y cuál es su función en el backend?

Una API (Interfaz de Programación de Aplicaciones) permite que diferentes aplicaciones se comuniquen entre sí. En el backend, las APIs exponen datos y funcionalidades para que el frontend o servicios externos los consuman.

2. ¿Cómo manejo la autenticación y autorización de usuarios?

Utilizo el sistema de autenticación de Django (django.contrib.auth) para gestionar el inicio de sesión, registro y permisos. Para las APIs, implemento autenticación con tokens o sesiones usando Django REST Framework, junto con permisos personalizados para controlar el acceso según roles de usuario.

3. ¿Qué es un ORM y cómo funciona en Django?

Un ORM (Object-Relational Mapping) permite interactuar con la base de datos usando modelos en lugar de SQL directo. En Django, el ORM convierte las consultas a SQL de manera automática.

4. ¿Cómo optimizo el rendimiento de las consultas a la base de datos?

Uso de select_related y prefetch_related en Django para evitar consultas repetitivas (N+1 problem). También implemento paginación, caching y optimización de índices en la base de datos.

5. ¿Cómo manejas errores y excepciones en un API?

Utilizo manejo de excepciones con bloques try-except y devuelvo respuestas con códigos HTTP adecuados (como 400 para errores del cliente o 500 para errores del servidor). En Django, DRF tiene clases de excepción como APIException.
